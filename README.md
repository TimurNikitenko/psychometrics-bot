# MBTI Psychometrics Bot - ИИ-бот для определения типа личности

Telegram-бот, который определяет тип личности по классической методике MBTI (16 типов). 

Главная особенность - нейросеть-классификатор обучалась на датасете живых постов с сабреддита `r/mbti`. Благодаря этому бот отлично понимает естественную рефлексию, рассуждения и форумный стиль общения. 

Бот умеет определять тип личности двумя способами: по свободному тексту и с помощью классического опросника.

## Возможности

- **ИИ-анализ текста (`/text`)** — пользователь делится своими мыслями, мнениями или историями (от 20 слов). Чем ближе стиль к постам на Reddit или форумах, тем точнее результат. Бот предсказывает тип и показывает процент совпадения по каждой из четырёх шкал (E–I, S–N, T–F, J–P).
- **Классический тест (`/test`)** — 70 вопросов опросника MBTI. Вопросы и ответы адаптированы под современный разговорный язык для комфортного прохождения.
- **Умное адаптивное тестирование** — если текст пользователя оказался многогранным и по некоторым шкалам результат пограничный (уверенность < 0.75), бот мягко предлагает «уточнить детали» и задаёт только те вопросы, которые относятся к спорным шкалам.
- **Два языка** — полноценная поддержка русского и английского языков (команда `/language`).
- **Автоперевод под капотом** — русский текст перед предсказанием автоматически переводится на английский (с помощью `Helsinki-NLP/opus-mt-ru-en`), так как модель лучше всего работает с англоязычным датасетом.

## Запуск

1. Зависимости: `pip install -r requirements.txt` (для модели также потребуется `requirements-ml.txt`).
2. `.env` должен быть со следующими переменными:
   - `BOT_TOKEN` — токен вашего бота, полученный у [@BotFather](https://t.me/BotFather);
   - `MBTI_MODEL_PATH` (опционально) — путь к модели (локальная папка или репозиторий Hugging Face, например `TimurNikitenko/mbti-classifier-ru`);
   - `ADMIN_ID` (опционально) — список ID администраторов через запятую.
3. Запустите бота: `python main.py`.

*Примечание: Если не указать `MBTI_MODEL_PATH`, режим `/text` будет недоступен, и бот предложит только классический тест.*

## Структура проекта

- `main.py` — точка входа, фоновая загрузка ML-моделей (чтобы бот отвечал мгновенно после запуска), старт поллинга.
- `handlers.py` — обработчики команд, логика машины состояний (FSM) для тестов и адаптивного уточнения.
- `config.py` — загрузка настроек из `.env`.
- `lexicon.py` — все текстовые интерфейсы, системные сообщения, описания типов и реакции бота.
- `mbti_classifier.py` — логика предсказания MBTI по тексту (загрузка весов, токенизация, инференс).
- `translator.py` — модуль машинного перевода (RU -> EN) для подготовки текста к анализу.
- `testing_utils.py` — база вопросов MBTI, логика подсчёта классического и адаптивного тестов.
- `model_upload.py` — утилита для выгрузки обученной модели в Hugging Face Hub.

## Работа с моделью

Модель классификатора можно подключить несколькими способами через `.env`:

- **Hugging Face Hub** — укажите repo id (например, `TimurNikitenko/mbti-classifier-ru`). Бот скачает веса из сабфолдера `mbti-classifier`.
- **Локальная директория** — укажите путь к папке с сохранёнными `model/` и `tokenizer/`.
- **Пустое значение** — загрузится базовый `distilbert-base-uncased` (подойдет только для технической проверки, так как веса не дообучены).
